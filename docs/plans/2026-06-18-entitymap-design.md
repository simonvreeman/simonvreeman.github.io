## 1. What EntityMap is (and its real status)

EntityMap is an open standard (CC BY 4.0) that publishes a structured, **entity-first** index of a site's knowledge for AI systems, RAG pipelines, and LLM apps. Tagline: *"Where sitemap.xml tells crawlers what pages exist, entitymap.json tells AI systems what a site knows"* â€” which entities a site covers, how they relate, and where the supporting evidence (chunks) lives.

- **Authors:** Fred Laurent and Dixon Jones (CTO / co-founder of InLinks and Waikay â€” single-vendor-originated).
- **Status:** v1.0 labelled "Stable / 2026-04-07", repo created 2026-05-27, in **public consultation until 2026-06-30, formal launch 2026-07-01**. ~27 GitHub stars, no confirmed third-party consumers yet.
- **Honest cost/benefit for this site:** No AI vendor consumes EntityMap today; vreeman.com already over-serves machine-readability via a live MCP server (`/mcp`), `schema.json`, and `llms.txt`. We are proceeding anyway as a **deliberate showcase / early-adopter implementation** (Simon is the target audience â€” an agent-readiness practitioner and technical-marketing consultant). The mitigation for the "low current value / maintenance drift" risk is: generate everything from one source, validate hard, and re-check after the July 1 freeze.

## 3. Verified spec facts the implementer must honor

Source of record is the GitHub `main` branch (`raw.githubusercontent.com/entitymap/entitymap/main/spec/v1.0/index.md` + `predicates.md`); the rendered `entitymap.org` page is a slightly stale build (shows 15 types + a `certification` field that the raw spec lacks).

**Two files, served from the DOMAIN ROOT, no auth:**
- `https://vreeman.com/entitymap.json` (machine-readable)
- `https://vreeman.com/entitymap.html` (human/crawler view; **MUST be generated from the JSON, never hand-maintained**; include plain-text attribution because many LLM pipelines strip HTML)
- There is **one canonical map per domain**. Sharding exists only for **>200 entities** (`shardOf` back-reference, split by size not meaning). Our ~160 stays unsharded.

**Required fields (validator errors if missing):**
- Root: `version` ("1.0"), `schema` ("https://entitymap.org/spec/v1.0"), `publisher` {`name`,`url`}, `generated` (ISO 8601, update every build), `entities` (â‰¥1).
- Entity: `entityId` (stable, never reused), `@type`, `name`, `description` (1â€“3 sentences), `hasChunks` (1â€“5). Plus `replacedBy` **if** status is `deprecated`/`merged`.
- Chunk: `chunkId` (unique), `text` (1â€“5 sentences, **â‰¤600 chars** â€” hard error), `sourceUrl` (public canonical URL), `pageTitle`, `publisher` (**MUST exactly equal root `publisher.name`** incl. case + spacing).

**Optional enrichment used here:** root `profile:"core"`, `verificationStatus:"self-declared"`, `vocabulary`; entity `sameAs` (Wikidata â€” SHOULD for Concepts), `alternateName`, `status`; chunk `retrieved`, `relevanceScore`, `contentType` (`definition`/`evidence`/`example`/`statistic`/`procedure`); relation `targetId`, `confidence`, `context`, `targetUri`.

**16 core `@type`s:** Concept, ProprietaryTerm, Methodology, Metric, Taxonomy Â· Person, Organization, SoftwareProduct, PhysicalProduct, Service, Platform, Place Â· Event, Standard, Regulation Â· Guide. (Custom types allowed if **namespaced**.)

**24 predicates (README's "23" is a confirmed typo):**
- Tier 1 â€” Hard (11, confidence not required): INSTANCE_OF, PART_OF, INCLUDES, DEPENDS_ON, REQUIRES, MEASURES, PRODUCED_BY, REGULATED_BY, AUTHORED_BY, AFFILIATED_WITH, COVERS.
- Tier 2 â€” Structural (7, confidence optional): RELATES_TO, PRECEDES, ENABLES, PREVENTS, CONFLICTS_WITH, DESCRIBED_BY, OFFERS.
- Tier 3 â€” Interpretive (6, **confidence REQUIRED** = `declared`|`inferred`): IMPROVES, DEGRADES, LEADS_TO, SUITED_FOR, TARGETS, ACHIEVES.
- Relation object: `predicate` (req), `targetName` (req), `targetId` (req for internal targets), `confidence` (req Tier 3).
- **Type-constrained predicates:** MEASURES (source must be Metric); AFFILIATED_WITH (source must be Person); COVERS (source must be Concept/ProprietaryTerm/Taxonomy); OFFERS (Organization source â†’ SoftwareProduct/Service/Platform/PhysicalProduct target).
- **No inverse pairs on the same entity pair:** PART_OFâ†”INCLUDES, ENABLESâ†”PREVENTS, IMPROVESâ†”DEGRADES, OFFERSâ†”PRODUCED_BY. No forbidden inverted forms (MEASURED_BY, ENABLED_BY, DESCRIBES, OFFERED_BY, â€¦) â€” swap subject/target instead.

**Custom vocabulary:** declared in root `vocabulary` block: `{ "predicates": ["TRANSLATED_BY"], "namespace": "https://vreeman.com/entitymap/vocab/v1" }`. Custom predicates MUST be uppercase, MUST NOT collide with the 24 standard names, and **MUST be documented at the declared namespace URI** (so that URL must resolve). Custom `@type`s must also be namespaced.

## 8. Generator, validation, discovery, serving

- `node entitymap/build.mjs`: stamp `generated`, assemble root (`version`, `schema`, `publisher`, `profile:"core"`, `verificationStatus:"self-declared"`, `vocabulary`, `entities`), **validate, refuse to write on any violation**, then emit `entitymap.json` (pretty) + `entitymap.html`.
- `validate.mjs` checks every rule in Â§3 (required fields; `@type` âˆˆ {16 âˆª custom}; â‰¤600-char chunks; exact publisher match; unique stable IDs; predicate âˆˆ {24 âˆª custom}; Tier-3 confidence; type-constrained predicates; inverse-pair guard; custom-predicate rules). `test/` proves good passes / broken fails (mirrors the 17-test approach used for `functions/mcp.js`).
- Discovery (minimal): add `EntityMap:` line to `robots.txt`; add `entitymap.html` to `sitemap.xml` (priority 0.9, weekly). **No edits to the 150 Stoic pages.**
- Serving: Cloudflare Pages serves both from root; verify `.json` â†’ `application/json`; optionally add a cache header in `_headers`.

5. **Sync** â€” `entitymap.html` is always regenerated; never hand-edit. `llms.txt` / `sitemap.xml` / MCP `CATALOG` remain parallel catalogs to keep roughly aligned.
